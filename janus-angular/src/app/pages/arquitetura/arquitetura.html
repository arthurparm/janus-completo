Resposta

O fluxo de processamento inicia com uma requisição do usuário que entra no Orquestrador (Llama 3.2). Este roteia dinamicamente para três caminhos especializados: o Cérebro Local (Phi-3, Code Llama) para processamento soberano, o Co-processador Nuvem (Gemini, OpenAI) para tarefas de alta complexidade, e o Sistema Nervoso (Tavily, APIs externas) para percepção do mundo digital. Todos os componentes interagem continuamente com o Sistema de Memória (Qdrant, Neo4j) antes de convergir para gerar a resposta final.

O Núcleo Cognitivo

O Cérebro (Local)

O núcleo de autonomia de Janus, garantindo operação contínua. Todos os modelos rodam localmente via Ollama.

  • Orquestrador (Llama 3.2 8B): Responsável pelo raciocínio de alto nível, decomposição de tarefas e roteamento dinâmico. (Ref: app/core/workers/orchestrator.py)
  • Professor/Curador (Phi-3-mini): Especialista em análise, validação de conhecimento e auto-reflexão. (Ref: app/core/optimization/reflexion_core.py)
  • Codificador (Code Llama 7B): Agente especializado para geração, análise e correção de código. (Ref: app/services/code_analysis_service.py)

O Co-processador (Nuvem)

Recursos de elite ativados pelo Roteador Dinâmico para tarefas de altíssima complexidade que excedem a capacidade local.

  • OpenAI (GPT-4o): Utilizado para planeamento de projetos complexos.
  • Google (Gemini 1.5 Pro): Utilizado para análise de grandes contextos ou modalidades mistas.
  • Grok & DeepSeek: Em avaliação para tarefas específicas de nicho.

Sistema de Memória Hierárquica

Memória de Trabalho (Curto Prazo)

Contexto imediato da conversa. Gerencia o estado atual da interação e os planos de curto prazo.

Tecnologia: Redis / Gerenciamento em memória (Stateful LangGraph)

Memória Episódica (Longo Prazo / Experiências)

Armazena experiências passadas, interações, erros e sucessos. É a base do aprendizado contínuo (Sprint 2).

Tecnologia: Qdrant (Banco de Dados Vetorial) (Ref: app/db/vector_store.py)

Memória Semântica (Longo Prazo / Fatos)

Um 'grafo de conhecimento' (Knowledge Graph) que armazena fatos extraídos, entidades e suas relações. Permite um entendimento profundo e estruturado do mundo.

Tecnologia: Neo4j (Ref: app/db/graph.py e app/core/memory/knowledge_graph_manager.py)

O Sistema Nervoso

Sentidos Periféricos (APIs Externas)

As ferramentas que permitem a Janus perceber e interagir com o mundo digital em tempo real.

  • Busca Web (Olhos): Tavily Search API
  • Contexto Local (Audição): APIs da HG Brasil (Clima/Finanças)
  • Análise de Linguagem (Tato): Google Cloud Natural Language API
  • Ferramentas Adicionais: Conjunto extenso definido em app/core/tools/agent_tools.py

Infraestrutura e Comunicação

Padrão de Comunicação: Orquestrador-Trabalhador

Janus não é um monólito. Ele usa uma arquitetura distribuída (Sprint 1) onde um Orquestrador central (orchestrator.py) decompõe tarefas e as distribui para 'Trabalhadores' especializados (workers/). A comunicação é assíncrona, garantindo resiliência e escalabilidade.

Tecnologias-Chave:

  • LangGraph: Orquestração de Estado e fluxos de trabalho complexos
  • RabbitMQ: Barramento de Mensagens para comunicação assíncrona (Ref: app/core/infrastructure/message_broker.py)
