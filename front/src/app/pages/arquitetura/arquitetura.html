O Janus opera como um ecossistema local de alta disponibilidade. Diferente de IAs na nuvem, ele roda inteiramente na sua infraestrutura (Docker). Sua "mente" é composta por uma orquestração de Agentes Especializados que interagem diretamente com seus arquivos, memória vetorial e modelos de linguagem locais.

🧬 COMPONENTES DO SISTEMA

>> NÚCLEO (Janus Core)

O centro de comando escrito em Python. É aqui que os Agentes Autônomos operam.

  • Orquestrador: Planeja tarefas complexas.
  • Coder: Escreve e edita código real.
  • Curator: Mantém a memória organizada.

>> MOTOR LOCAL (Ollama)

A força de inteligência. Roda modelos avançados (como Llama 3) diretamente na sua placa de vídeo.

  • > Processamento 100% Offline e Privado.
  • > Sem custos de API externa.
  • > Alta velocidade de inferência (GPU).

>> MEMÓRIA HÍBRIDA

O Janus lembra do que você fez usando dois tipos de bancos de dados.

  • Qdrant (Vetores): Entende o "significado" e contexto do código.
  • Neo4j (Grafo): Mapeia como os arquivos se conectam (importações, classes).

>> INTERFACE VISUAL

O painel de controle. Construído em Angular.

  • > Design Moderno "Glassmorphism".
  • > Comunicação via API REST / Polling.
  • > Visualização de Arquivos e Metas.

Neo4j & Filesystem

O Agente executa a ação e salva o resultado no Grafo de Conhecimento. Task marcada como COMPLETED.

🔄 OTIMIZAÇÃO CONTÍNUA (Reflexion)

Ciclo de Aprendizado

O Janus não apenas executa código; ele aprende com seus erros usando um mecanismo de Reflexão. Quando uma tarefa falha, ele entra em um ciclo de análise para corrigir o problema.

  1. TENTATIVA: O Agente executa uma ação (ex: escreve um código).
  2. AVALIAÇÃO: O sistema analisa o resultado (Testes, Linters, Erros).
  3. REFLEXÃO: O Modelo gera uma crítica sobre a falha.
  4. MEMÓRIA: A lição é salva para o futuro.
  5. NOVA TENTATIVA: O Agente tenta novamente com o novo contexto.
