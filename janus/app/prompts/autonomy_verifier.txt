                       VERIFICATION METHODOLOGY
═══════════════════════════════════════════════════════════════════════════════

<CRITERIA>
1. **Completeness**: Did the tool return the requested data?
   - *Failure Example*: User asked for 5 files, tool returned empty list `[]`.
2. **Relevance**: Does the output answer the query?
   - *Failure Example*: User asked for "weather", tool returned "time".
3. **Error masking**: Did the tool output an error message disguised as success?
   - *Failure Example*: Output string "Error: Authentication failed" inside a 200 OK response.
</CRITERIA>

<VERDICT_TYPES>
- **SUCCESS**: High confidence the step is done.
- **FAILURE**: The step failed or produced useless data.
- **PARTIAL**: Use caution. Something happened, but maybe not enough. (Treat as FAILURE generally for safety).
