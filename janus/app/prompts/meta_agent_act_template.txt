┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 1: Categorization                                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│ Group lessons by theme:                                                     │
│ ✓ Errors and failures                                                      │
│ ✓ Performance degradation                                                  │

│ ✓ Integration problems                                                     │
│ ✓ Identify frequency of each category                                      │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 2: Pattern Identification                                              │
├─────────────────────────────────────────────────────────────────────────────┤
│ Find recurring patterns:                                                    │
│ ✓ Errors appearing multiple times                                          │
│ ✓ Progressive degradations                                                 │
│ ✓ Correlations between different failure types                             │
│ ✓ Temporal sequences (X always precedes Y)                                 │
│ ✓ Outliers or anomalies                                                    │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 3: Root Cause Analysis                                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│ For each pattern, ask:                                                      │
│ • Why did this happen?                                                     │
│ • Which component/process failed?                                          │
│ • Were there warning signs?                                                │
│ • What is the fundamental cause (not just the symptom)?                    │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 4: Impact Assessment                                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│ Evaluate each problem:                                                      │
│ ✓ Severity: How critical is it?                                            │
│ ✓ Frequency: How often does it occur?                                      │
│ ✓ Scope: How many users/systems affected?                                  │
│ ✓ Trend: Getting better or worse?                                          │
└─────────────────────────────────────────────────────────────────────────────┘
