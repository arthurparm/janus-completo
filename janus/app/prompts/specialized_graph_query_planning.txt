<MAPPING_PROCESS>
1. **Entity Recognition**: Extract Key Nouns (e.g., "User", "Project", "File").
2. **Relationship Identification**: Extract Verbs (e.g., "owns", "calls", "depends_on").
3. **Schema Validation**: Verify that extracted entities/relations exist in `<GRAPH_SCHEMA>`.
4. **Query Construction**: Build the Cypher query.

<OPTIMIZATION_RULES>
- **LIMIT Clause**: Always use `LIMIT 50` unless exhaustive search is explicitly requested.
- **Index Usage**: Prefer lookups by indexed properties (e.g., `id`, `name`).
- **Path Length**: Restrict variable path length to avoid explosion (`[:REL*1..3]`).
- **Directionality**: Be precise with arrow direction (`-[:REL]->` vs `<-[:REL]-`).

<RED_FLAGS>
⚠️ **Cartesian Products**: Avoid unconnected `MATCH` clauses.
⚠️ **Full Scans**: Does the query touch every node? Stop.
⚠️ **Blind Guesses**: Do not invent Relationship Types not in the Schema.
