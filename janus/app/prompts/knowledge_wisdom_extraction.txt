<EXTRACTION_PROTOCOL>

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 1: Content Analysis                                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│ Analyze the raw text:                                                       │
│ ✓ Identify key topics and themes                                           │
│ ✓ Recognize entities (people, tech, concepts)                              │
│ ✓ Detect cause-effect relationships                                        │
│ ✓ Spot decision points and outcomes                                        │
│ ✓ Find  patterns or recurring elements                                      │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 2: Knowledge Classification                                            │
├─────────────────────────────────────────────────────────────────────────────┤
│ Categorize extracted knowledge:                                             │
│ • **Facts**: Objective, verifiable information                             │
│ • **Patterns**: Recurring themes or behaviors                              │
│ • **Lessons**: "If X, then Y" causal wisdom                                │
│ • **Concepts**: Abstract ideas or principles                               │
│ • **Relationships**: Connections between entities                          │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 3: Wisdom Distillation                                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│ Transform information into actionable wisdom:                               │
│ ✓ Generalize from specific instances                                       │
│ ✓ Extract underlying principles                                            │
│ ✓ Identify when/where knowledge applies                                    │
│ ✓ Note confidence level and evidence strength                              │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 4: Structured Representation                                           │
├─────────────────────────────────────────────────────────────────────────────┤
│ Format knowledge for storage:                                               │
│ ✓ Use consistent schema                                                    │
│ ✓ Add metadata (source, timestamp, confidence)                             │
│ ✓ Link to related knowledge                                                │
│ ✓ Include examples or evidence                                             │
└─────────────────────────────────────────────────────────────────────────────┘

                      EXTRACTION PRINCIPLES
═══════════════════════════════════════════════════════════════════════════════

<PRINCIPLES>

⚠️ **Ground in Text**: Extract only what's actually in the text  
⚠️ **Don't Hallucinate**: Never invent facts or relationships  
⚠️ **Be Specific**: Vague wisdom is useless—be concrete  
⚠️ **Preserve Context**: Include "when/where" for applicability  
⚠️ **Assess Confidence**: Not all knowledge is equally certain  
⚠️ **Cite Evidence**: Link back to source text  
⚠️ **Tag Thoroughly**: Good tags enable future retrieval  

</PRINCIPLES>

□ Confidence levels assigned appropriately
□ Evidence/quotes included for verification
□ Tags are relevant and descriptive
□ JSON output is valid and complete
