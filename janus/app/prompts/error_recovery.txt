├─────────────────────────────────────────────────────────────────────────────┤

│ 2. Why did it happen? [Proximate cause]                                    │

├─────────────────────────────────────────────────────────────────────────────┤

│ ✓ Update system knowledge for future                                       │

                         RECOVERY PRINCIPLES
═══════════════════════════════════════════════════════════════════════════════

<PRINCIPLES>

⚠️ **Fail Fast, Recover Smart**: Don't hide errors, handle them explicitly  
⚠️ **Isolate Failures**: Prevent cascading failures  
⚠️ **Preserve State**: Don't lose critical data during recovery  
⚠️ **Log Everything**: Recovery attempts are valuable learning data  
⚠️ **Timeout Retries**: Never retry indefinitely  
⚠️ **Exponential Backoff**: Space out retry attempts  
⚠️ **Circuit Breakers**: Stop trying if failing repeatedly  
⚠️ **Idempotent Operations**: Safe to retry without side effects  

</PRINCIPLES>

<COMMON_STRATEGIES>

| Error Type | Best Strategy | Example |
|------------|---------------|---------|
| Network timeout | RETRY with backoff | 3 attempts: 1s, 2s, 4s |
| Resource unavailable | FALLBACK | Use cache instead of DB |
| Invalid input | DEGRADE | Skip validation, log warning |
| Component crash | ISOLATE + RESTART | Restart worker, quarantine job |
| Data corruption | ESCALATE | Alert admin, stop processing |

</COMMON_STRATEGIES>
