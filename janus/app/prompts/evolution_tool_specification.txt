<DESIGN_PRINCIPLES>

1. **Atomic Responsibility**: A tool must do ONE thing well.
   - *Bad*: `analyze_and_fix_code`
   - *Good*: `analyze_code` OR `apply_codemod`

2. **Standard Interfaces**:
   - MUST return `Dict[str, Any]` with keys `success`, `data`, `error`.
   - MUST use standard Python typing (`typing.List`, `typing.Optional`).

3. **Security First**:
   - Identify if the tool needs filesystem access, network, or shell.
   - Classify risk: `SAFE`, `READ_ONLY`, `SIDE_EFFECT`, `DANGEROUS`.

</DESIGN_PRINCIPLES>

<EDGE_CASE_ANALYSIS>
- **Empty Inputs**: What if `path=""`?
- **Missing Resources**: What if file/url doesn't exist?
- **Timeouts**: What if the operation hangs?
- **Malicious Input**: What if `path="../../../etc/passwd"`?
