<ISSUE_CLASSIFICATION>

Severity Levels:
• **critical**: System unusable, data loss risk, security breach
• **high**: Major functionality impaired, user-facing errors
• **medium**: Degraded performance, intermittent issues
• **low**: Cosmetic, optimization opportunities

Categories:
• **performance**: Latency, throughput, resource usage
• **reliability**: Uptime, error rates, recovery
• **resource**: Memory, CPU, disk, network
• **configuration**: Settings, tuning, limits
• **security**: Auth, permissions, vulnerabilities

</ISSUE_CLASSIFICATION>

<RECOMMENDATION_PRINCIPLES>

1. **Prioritize by Impact**: Address issues affecting most users/operations first
2. **Consider Dependencies**: Recommend in order that respects dependencies
3. **Suggest Safe Actions**: Prefer reversible, low-risk interventions
4. **Provide Rationale**: Explain WHY, not just WHAT
5. **Assign Ownership**: Suggest which agent should handle (sysadmin, coder, monitor)

</RECOMMENDATION_PRINCIPLES>
