<COMPRESSION_PROTOCOL>

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 1: Information Extraction                                              │
├─────────────────────────────────────────────────────────────────────────────┤
│ Identify and categorize all information:                                    │
│ ✓ Critical facts (decisions, agreements, key data)                         │
│ ✓ Important context (goals, preferences, constraints)                      │
│ ✓ Moderate relevance (supporting details)                                  │
│ ✓ Low relevance (pleasantries, redundant info)                             │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 2: Priority-Based Filtering                                            │
├─────────────────────────────────────────────────────────────────────────────┤
│ Apply retention rules:                                                      │
│ • MUST KEEP: Decisions, commitments, unresolved questions                  │
│ • SHOULD KEEP: Goals, preferences, context that affects future responses   │
│ • OPTIONAL: Background details, explanations, examples                     │
│ • DISCARD: Greetings, confirmations, redundant statements                  │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 3: Semantic Compression                                                │
├─────────────────────────────────────────────────────────────────────────────┤
│ Compress retained information:                                              │
│ ✓ Merge redundant statements                                               │
│ ✓ Use concise phrasing (active voice, direct statements)                   │
│ ✓ Replace examples with generalizations when possible                      │
│ ✓ Preserve specific data (numbers, names, dates) exactly                   │
│ ✓ Maintain temporal order where relevant                                   │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 4: Verification                                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│ Ensure quality:                                                             │
│ ✓ No critical information lost                                             │
│ ✓ Target length achieved                                                   │
│ ✓ Summary is coherent and readable                                         │
│ ✓ Specific details (names, numbers) are accurate                           │
└─────────────────────────────────────────────────────────────────────────────┘

</COMPRESSION_PROTOCOL>
