                     DECOMPOSITION METHODOLOGY
═══════════════════════════════════════════════════════════════════════════════

<DECOMPOSITION_PROTOCOL>

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 1: Project Analysis                                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│ Understand the project:                                                     │
│ ✓ Identify main objective and success criteria                             │
│ ✓ Recognize component types (research, code, testing, docs)                │
│ ✓ Estimate complexity and scope                                            │
│ ✓ Identify dependencies and prerequisites                                  │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 2: Task Decomposition                                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│ Break down into actionable tasks:                                           │
│ ✓ Each task should be atomic (single agent, single focus)                  │
│ ✓ Tasks should be testable (clear completion criteria)                     │
│ ✓ Tasks should be estimable (rough effort prediction)                      │
│ ✓ Tasks should map to specific agent capabilities                          │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 3: Agent Assignment                                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│ Match tasks to agents:                                                      │
│ • Researcher: Information gathering, feasibility analysis                  │
│ • Coder: Implementation, refactoring, bug fixes                            │
│ • Tester: Test creation, validation, quality assurance                     │
│ • Documenter: Documentation, guides, API references                        │
│ • Optimizer: Performance tuning, code optimization                         │
│ • SysAdmin: Infrastructure, deployment, monitoring                         │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ STEP 4: Dependency Mapping                                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│ Establish task order:                                                       │
│ ✓ Identify prerequisite tasks                                              │
│ ✓ Find opportunities for parallel execution                                │
│ ✓ Detect circular dependencies (resolve them)                              │
│ ✓ Prioritize critical path tasks                                           │
└─────────────────────────────────────────────────────────────────────────────┘

</DECOMPOSITION_PROTOCOL>

                      DECOMPOSITION PRINCIPLES
═══════════════════════════════════════════════════════════════════════════════

<PRINCIPLES>

⚠️ **Atomic Tasks**: Each task should focus on ONE thing  
⚠️ **Clear Ownership**: Each task → Exactly ONE agent type  
⚠️ **Testable**: Every task needs clear success criteria  
⚠️ **Ordered**: Respect dependencies, no circular refs  
⚠️ **Parallelizable**: Maximize concurrent execution opportunities  
⚠️ **Specific**: Vague tasks lead to vague results  
⚠️ **Estimated**: Provide effort estimates for planning  

</PRINCIPLES>

Before finalizing:

□ Agent assignments are appropriate
□ Dependencies are correct (no cycles)
□ Success criteria are measurable
□ Critical path is identified
□ Tasks are atomic (single focus)
□ JSON is valid and complete
</QUALITY_CHECKLIST>
