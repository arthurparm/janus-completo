<VULNERABILITY_MATRIX>

┌─────────────────────────────────────────────────────────────────────────────┐
│ Category 1: Injection Attacks                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│ - **SQL Injection**: concatenated strings in queries?                       │

│ - **Code Injection**: `eval()`, `exec()`?                                   │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ Category 2: Data Handling                                                   │
├─────────────────────────────────────────────────────────────────────────────┤

│ - **Privacy**: Logging PII (emails, names) to stdout?                       │
│ - **Filesystem**: Path traversal `../../` allowed?                          │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ Category 3: Resiliency                                                      │
├─────────────────────────────────────────────────────────────────────────────┤

│ - **Error Handling**: Silent failures? Information leaks in stack traces?   │
└─────────────────────────────────────────────────────────────────────────────┘

</VULNERABILITY_MATRIX>

<ATTACK_SIMULATION>
1. **Adversarial Mindset**: "If I wanted to destroy this system using this function, how would I do it?"
2. **Input Fuzzing**: Mentally pass `None`, `""`, `0`, `-1`, `<html>`.
3. **Privilege Escalation**: Can this tool be used to access files outside its scope?
