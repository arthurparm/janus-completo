<PRESERVATION_RULES>

┌─────────────────────────────────────────────────────────────────────────────┐
│ MUST PRESERVE (Signal)                                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│ ✓ **Decisions**: Final outcomes of detailed discussions.                    │

│ ✓ **Errors**: Specific error messages encountered (for history tracking).   │

│ ✓ **Code**: Only high-level architectural changes, not full snippets.       │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ MUST DISCARD (Noise)                                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│ ✗ **Pleasantries**: "Hello", "Thank you", "You're welcome".                 │
│ ✗ **Repetitive Retries**: "Trying X... Failed. Trying Y... Failed."         │
│   -> Abstract to: "Tried X and Y, both failed."                             │
│ ✗ **Intermediate Thoughts**: The agent's internal monologue (Chain of Thought). │
└─────────────────────────────────────────────────────────────────────────────┘

</PRESERVATION_RULES>

<COMPRESSION_TECHNIQUES>
1. **Entity Extraction**: Identify core nouns (Files, Servers, Users).
2. **State Snapshotting**: Describe the system state *now*, not the history of how it got there.
3. **Causal Linking**: "Because X failed, we switched to Y."

<RED_FLAGS>
⚠️ **Lossy Compression**: Deleting a critical requirement (e.g., "Must be Python 3.9") is unacceptable.
⚠️ **Ambiguity**: "We fixed the bug" -> Bad. "We fixed the NullPointer in auth.ts" -> Good.
⚠️ **Hallucination**: Do not invent success if the last state was failure.
