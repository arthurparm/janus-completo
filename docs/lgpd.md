# LGPD Review

## Checklist
- [x] Check for PII logging in services
- [x] Check for full message content logging in commands/event publishers
- [x] Check voice command logging in daemon
- [x] Check log rotation and purging policies
- [x] Check in-memory state or temporary storage for PII leakage

## Gaps Found

1. **PII Logging in `ChatCommandHandler`**: The `ChatCommandHandler` service logs full user feedback content in `args`, which constitutes a PII risk.
2. **Message Preview Logging in `ChatEventPublisher`**: The `ChatEventPublisher` service (`backend/app/services/chat_event_publisher.py`) logs message content previews in `_publish_to_log`, which constitutes a PII logging risk.
3. **Productivity Tools PII Logging**: `backend/app/core/tools/productivity_tools.py` logs email metadata (recipient, subject), which constitutes a PII risk, though the body content is not logged.
4. **Daemon Voice Logs**: The Janus Daemon (`backend/app/interfaces/daemon/daemon.py`) processes voice commands via `ChatService` and logs voice command inputs, constituting a documented LGPD/PII risk.
5. **Lack of Log Rotation/Purging**: Application logs (`janus.log`) generated by `backend/main.py` lack automated rotation and audit purging policies, posing storage and LGPD compliance risks.
6. **In-Memory Storage Leak Risk**: `backend/app/core/tools/productivity_tools.py` uses global in-memory dictionaries (`_notes`, `_calendar_events`) for storage, which creates a non-persistent data leak risk.

## Actionable Recommendations

- **Sanitize Logging in Chat Services**: Implement redaction/sanitization logic in `ChatCommandHandler` and `ChatEventPublisher` before logging `args` and `payload` to ensure no PII is inadvertently logged.
- **Remove/Redact Productivity Metadata Logs**: Strip sensitive email addresses and subjects from logging statements in `productivity_tools.py`. Replace explicit addresses with hashed versions or simply count parameters.
- **Disable/Redact Voice Logs**: Remove the explicit text logging of recognized commands in `daemon.py` or apply a strict PII masking before logging.
- **Implement Log Rotation**: Add automated rotation and retention purging policies to `backend/main.py` or the central logging configuration. Set limits on how long log lines containing user actions are retained (e.g., 30 days).
- **Remove In-Memory Dictionaries**: Refactor `productivity_tools.py` to use a persistent, access-controlled data store instead of global memory variables.
