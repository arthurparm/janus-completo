# Janus AI Architect

Janus AI Architect is a monorepo project featuring a sophisticated AI agent system. It is composed of an **Angular 20 Frontend** and a **FastAPI Backend** (Python), integrating advanced memory architectures (Bicameral Memory), RAG (Retrieval-Augmented Generation), and autonomous agent capabilities.

## Project Structure

- **[front/](front/)**: The Angular frontend application (SPA).
- **[janus/](janus/)**: The Python FastAPI backend, including the agent engine, memory systems, and workers.
- **[docs/](docs/)**: Comprehensive documentation generated by the project workflow.
- **[ROADMAP.md](ROADMAP.md)**: The project roadmap and backlog.

## Architecture Summary

Janus utilizes a "Bicameral" memory architecture:
- **Fast Thinking (System 1)**: Powered by **Qdrant** for episodic and vector memory.
- **Slow Thinking (System 2)**: Powered by **Neo4j** for semantic and knowledge graph memory.

The system is designed with a layered architecture, separating the kernel, services, repositories, and API routes. It employs a Meta-Agent architecture using LangGraph to manage state and autonomy.

For detailed architecture documentation, please refer to the [Documentation Index](docs/index.md).

## Quick Start

### Prerequisites

- **Docker** and **Docker Compose**
- **Node.js 20+** (for local frontend)
- **Python 3.11+** (for local backend)

### Running with Docker (Recommended)

To start the entire stack (Frontend, Backend, Databases, Message Broker):

```bash
docker compose up -d
```

### Running Locally

#### Backend (`janus`)

```bash
cd janus
# Create a virtual environment (optional but recommended)
python -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Start the server
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

The backend API will be available at `http://localhost:8000`.

#### Frontend (`front`)

```bash
cd front
npm install
npm start
```

The frontend will be available at `http://localhost:4200`. It is configured to proxy API requests to `http://localhost:8000`.
