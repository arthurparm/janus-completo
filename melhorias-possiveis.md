# Janus - Melhorias Possiveis

Data de criacao: 2026-02-12
Objetivo: centralizar ideias de evolucao do Janus em um unico backlog vivo, para manter organizacao nas conversas futuras.

## Como usar este documento

1. Cada item deve ter dono, prioridade e status.
2. Priorizar por impacto em usuario + reducao de risco tecnico.
3. Quebrar itens grandes em historias pequenas e testaveis.
4. Revisar semanalmente e mover itens para roadmap ativo.

## Escala sugerida

- Prioridade: P0 (urgente), P1 (alta), P2 (media), P3 (baixa)
- Esforco: S (1-2 dias), M (3-7 dias), L (2+ semanas)
- Status: ideia, planejado, em andamento, concluido, descartado

### [SG-020] Mitigar vulnerabilidade de injeção SQL no dedupe_service
- Problema atual: O uso de f-strings para construir a query `UPDATE {table}` foi sinalizado pelo Bandit (B608) como vetor potencial de injeção SQL.
- Solucao proposta: Remover construção por f-strings, migrando para sintaxe nativa com parâmetros de update do SQLAlchemy ou validação rígida de metadados se o nome da tabela for dinâmico.
- Impacto esperado: Prevenir vulnerabilidades de banco de dados.
- Riscos: Redução de performance caso consultas complexas passem por camadas adicionais.
- Dependencias: SQLAlchemy
- Prioridade: P1
- Esforco: S
- Dono: a definir
- Status: ideia

### [SG-021] Substituir parser XML vulnerável no document_parser_service
- Problema atual: O pacote nativo `xml.etree.ElementTree` está exposto a ataques XML (XXE / Billion Laughs) se injetarem XMLs maliciosos (Bandit B314 e B405).
- Solucao proposta: Instalar e adotar a biblioteca segura `defusedxml` para parsing das estruturas de dados.
- Impacto esperado: Proteção total contra Entity Expansion.
- Riscos: Nenhum.
- Dependencias: defusedxml
- Prioridade: P1
- Esforco: S
- Dono: a definir
- Status: ideia

### [SG-022] Risco de OS Command Injection em ferramentas do Semantic Commit
- Problema atual: O `subprocess.run(cmd)` confia no comando repassado com shell ativo sem proteção total contra injeções. (Bandit B603).
- Solucao proposta: Restringir a função para passar apenas argumentos explícitos (`["git", "commit", ...]`) invés de concatenar comandos.
- Impacto esperado: Proteção contra RCE.
- Riscos: Nenhum.
- Dependencias: subprocess
- Prioridade: P2
- Esforco: S
- Dono: a definir
- Status: ideia

### [SG-023] Proteção do Endpoint do Windows Agent
- Problema atual: O app FastAPI no Windows Host está escutando na porta 5001 expondo recursos críticos do SO sem Auth.
- Solucao proposta: Adicionar um `API_KEY` injetado pelo Container na subida do Docker, que deve estar contido em cada Header da requisição.
- Impacto esperado: Garantir zero trust no container host communication.
- Riscos: Containers antigos precisarem ser reconstruídos.
- Dependencias: fastapi middleware
- Prioridade: P0
- Esforco: S
- Dono: a definir
- Status: ideia

### [SG-024] Atualização de Dependências Críticas (NPM e PIP)
- Problema atual: A rotina diária encontrou pacotes vulneráveis como pip (CVE-2026-1703) e no NPM (`dompurify`, `hono`, `immutable`, `tar`).
- Solucao proposta: Alterar as tags de versão, atualizar dependências e aplicar `npm audit fix --force`.
- Impacto esperado: Remover exploits conhecidos das cadeias de build e interface do cliente.
- Riscos: Dependência quebrar o build.
- Dependencias: pip, npm
- Prioridade: P1
- Esforco: S
- Dono: a definir
- Status: ideia
